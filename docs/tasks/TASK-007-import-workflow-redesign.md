## 目标

把数据导入页优化成清晰的三步任务流。重点是字段映射、缺字段提示、手动年度/期间、错误结果展示更清楚。

## 依赖

- 必须等待 `TASK-005` 完成。
- 先阅读 `docs/UI_OPTIMIZATION_PLAN.md`。

## 允许修改范围

可以修改：

- `frontend/src/views/DataImportView.vue`
- `frontend/src/types/index.ts`（仅限前端展示类型，不得改 API 契约）

不要修改：

- 后端文件
- `frontend/src/views/HomeView.vue`
- `frontend/src/views/CompaniesView.vue`

## 具体要求

1. 步骤 1：
   - 左侧为公司、数据类型、会计年度、会计期间。
   - 右侧为上传区域和文件要求。
   - 当文件缺年度/期间时，手动输入提示要更明确。
2. 步骤 2：
   - 字段映射表更紧凑。
   - 右侧增加“导入前检查”面板：缺失字段、已忽略列、手动期间状态。
   - 已映射/未映射/已忽略状态清晰，但不要大面积彩色背景。
3. 步骤 3：
   - 区分系统错误、行级错误、部分成功。
   - 错误列表可扫读。
   - 提供返回修改映射和再次导入路径。
4. 交互：
   - 不破坏当前 API 请求。
   - `column_mapping` 仍为 `{ 原始表头: 标准字段 }`。
