# 审计系统 UI 优化计划

日期：2026-06-18

## Product Design Brief

- 产品：审计系统基座，面向审计人员、项目经理、数据导入人员。
- 当前问题：界面过于普通、松散、缺少专业审计系统的秩序感；首页、导入向导和表格页面视觉层级不够明确。
- 目标风格：专业、克制、密集可扫读的企业级 SaaS 操作台。不要营销页，不要花哨插画，不要大面积渐变，不要把每个区块都做成漂浮卡片。
- 交互等级：保持现有 Vue/Element Plus 功能完整，优化视觉、布局、状态和信息组织。
- 视觉来源：当前项目代码、现有 Element Plus 组件、`frontend/src/styles/tokens.css` 与业务语义。

## 设计修订：从后台模板升级为审计工作台

用户反馈：上一版左侧任务栏偏老土，首页“各种看板按钮”不够先进。新的方向不再做传统侧边栏 + 大按钮看板，而是做“审计工作台”：

1. 左侧从 220px 老式菜单升级为“双层导航”：56px 窄图标轨道 + 168px 工作区面板。
2. 首页从按钮看板升级为“工作概览”：顶部 搜索栏、任务队列、待处理事项、最近导入。
3. 快捷入口从大卡片改为“操作入口 + 智能推荐动作”，像专业数据产品，不像后台模板。
4. 视觉允许更先进，但必须克制：深色轨道、冷灰内容区、蓝青色数据线、低饱和状态色；不要霓虹泛滥、不要游戏 HUD、不要科幻装饰。

## 总方向

把当前界面从“通用后台模板”升级为“审计数据工作台”：

1. 左侧导航更像现代数据产品：窄图标轨道负责全局模块，工作区面板负责当前模块内导航。
2. 顶部栏从简单面包屑升级为命令上下文栏：当前账期、全局搜索/命令入口、当前任务状态。
3. 首页从欢迎页升级为审计工作台：关键指标、待处理事项、最近最近导入、常用操作。
4. 数据导入页从表单堆叠升级为清晰任务流：上传、字段映射、校验结果、错误修复路径。
5. 公司管理页从普通增删改查表升级为数据管理表：搜索、筛选、状态、行操作、空状态一致。

## 设计原则

### 信息密度

- 审计软件优先“看得清、找得到、批量处理快”，不是展示型大留白。
- 页面最大宽度保持可控，但表格和导入向导要充分利用横向空间。
- 统计信息用紧凑 metric strip，不用过大的营销式数字卡。

### 色彩

- 主色继续使用蓝色，但降低饱和度，偏专业蓝灰。
- 保留语义色：成功、警告、危险、信息。
- 页面背景从纯灰改为更细微的冷灰，减少“模板感”。
- 风险、错误、缺字段等状态用色要节制，不能满屏红黄。

### 结构

- 少用阴影，多用边框、分隔线、行高、间距建立层级。
- 卡片只用于真正的容器：统计组、数据表、导入步骤，不做卡片套卡片。
- 表格操作区固定但不要拥挤，按钮用文字链接 + 图标，危险操作保持确认。
- 不再使用“几个大按钮卡片”作为首页核心入口；操作入口改为搜索栏、任务队列和操作建议。

### 字体与尺寸

- 正文 14px，辅助信息 12-13px，页面标题 20-22px。
- 数字使用 tabular nums，金额和条数右对齐。
- 禁止负 letter-spacing，中文界面保持默认字距。

## 目标界面结构

### 全局 App Shell

- 左侧：56px 暗色图标轨道 + 168px 工作区面板。轨道只显示模块图标和短标签，工作区面板显示当前模块下的二级导航、项目状态和版本。
- 当前项：不用传统整块蓝底，改用发光左边线、细描边和轻微 surface tint。
- 顶部高度：56px，白底细边线，显示页面标题、当前账期、全局 搜索栏、时间。
- 主内容：冷灰背景，内容区 1280-1360px，保持 24px 外边距。
- 移动端：侧边栏收窄，核心页面表格横向滚动，操作按钮堆叠。

### 首页

首页改成“审计工作台”：

- 顶部：页面标题 + 当前工作期间 + 搜索栏（例如“搜索单位或功能 / 搜索单位 / 导入文件”）。
- 第一行：4 个紧凑指标，使用低饱和趋势线和状态点，不使用大渐变图标块。
- 主区左侧：最近导入，显示“上传 → 映射 → 校验 → 入库”的最近批次状态。
- 主区右侧：待处理事项，列出缺字段、借贷不平、待复核导入。
- 底部：常用操作，用操作入口呈现，例如“导入序时账”“补全字段映射”“新增被审计单位”，不做大按钮看板。

### 数据导入页

数据导入页改成“导入任务向导”：

- 顶部：步骤条保持，但改为更紧凑的进度轨道。
- 步骤 1：左侧选择公司/类型/期间，右侧上传区域和文件摘要。
- 步骤 2：字段映射分左右：
  - 左侧字段映射表：原始列、系统字段、状态、示例值。
  - 右侧校验面板：缺失字段、手动期间、导入前检查。
- 步骤 3：结果页显示成功、失败、错误分类和下载错误明细入口。

### 公司管理页

公司管理页改成“单位管理管理”：

- 顶部搜索栏：关键字、状态筛选、行业筛选、新建单位。
- 表格：去掉编号主显示，改显示单位名称、编码、税号、行业、状态、创建时间、操作。
- 行状态：启用/停用用低饱和标签。
- 空状态：明确引导“新建单位”。
- 表单：分组更清晰，必填项更明确。

## 总体验收标准

1. `cd frontend; npm run build` 通过。
2. 三个页面 `/`、`/data/import`、`/data/companies` 没有明显溢出、遮挡、错位。
3. 1366px 桌面宽度可用，480px 窄屏不阻断核心操作。
4. 视觉风格一致：颜色、间距、圆角、边框、表格、按钮状态一致。
5. 不破坏当前前后端契约，不改后端业务逻辑。
